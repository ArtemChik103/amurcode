# Предметные правила MVP

## Шаблоны

- `all` - все данные без предметного ограничения.
- `kik` - КЦСР содержит `975` или `978` с 6-й позиции: `object_code_norm[5:8] in {"975", "978"}`. В текущих данных кейса КИК представлен кодами `975xx`.
- `skk` - КЦСР содержит `6105` с 6-й позиции: `object_code_norm[5:9] == "6105"`.
- `two_thirds` - КЦСР содержит `970` с 6-й позиции: `object_code_norm[5:8] == "970"`.
- `okv` - капитальные вложения по КВР из набора `400`, `410`-`417`.

## Показатели

- Лимиты берутся из РЧБ как состояние на отчетную дату.
- БО берутся из РЧБ как состояние на отчетную дату.
- Касса берется из РЧБ как состояние на отчетную дату.
- Соглашения берутся из выгрузки соглашений как состояние на отчетную дату.
- Контракты и платежи берутся из выгрузки госзаказа как события с накоплением до даты.
- БУ/АУ берутся из выгрузки БУАУ как события с накоплением до даты.

В РЧБ поля лимитов, БО и кассы ищутся по префиксам `Лимиты ПБС`, `Подтв. лимитов по БО`, `Всего выбытий`. Год в названии колонки не является частью правила, поэтому `... 2025 год` и `... 2026 год` обрабатываются одинаково.

## Цепочка денег

В пользовательской таблице используется цепочка:

- `plan = limit + obligation`
- `documents = agreement + contract`
- `paid = payment + buau`
- `cash = cash`

Проблемные статусы:

- `no_documents` - есть план, нет соглашений и контрактов.
- `no_payments` - есть документы, нет оплат.
- `no_cash` - есть план, касса равна нулю.
- `low_cash` - касса меньше 25% от плана.
- `data_gap` - объект есть только в одном источнике.

## Риск

Риск - это подсказка для приоритизации проверки, а не юридический вывод. Балл помогает руководителю сначала открыть объекты, где одновременно есть проблема исполнения и крупная сумма.

Формула ограничена 100 баллами:

- +30, если нет кассового исполнения;
- +25, если есть документы, но нет оплат;
- +20, если есть план, но нет документов;
- +15, если касса ниже 25% от плана;
- +15, если данные есть не во всех источниках;
- +10, если план от 1 млрд;
- +7, если план от 100 млн;
- +5, если есть документы, но оплачено 0;
- +5, если касса 0 при плане больше 0.

Уровни: `critical` от 75, `high` от 50, `medium` от 25, `low` ниже 25. В UI они показываются как `Критичный`, `Высокий`, `Средний`, `Низкий`.

График `Риски` группирует объекты по `risk_level`: `critical >= 75`, `high >= 50`, `medium >= 25`, `low < 25`. Под графиком есть раскрываемое объяснение с человекочитаемыми причинами риска: нет кассы, нет оплат, нет документов, низкая касса, разрыв источников, крупный план. Риск не является юридическим выводом; это управленческий индикатор для выбора первоочередной проверки.

## Бюджеты

Областной и муниципальные бюджеты пока используются как строковое поле `budget` из источника. Отдельное поле `budget_scope` не выделено.

## Даты

- Основной режим - `as_of`, состояние на дату.
- `snapshot` - дата месячного среза источника.
- `event_date` - дата операции, если она есть в источнике.
- `document_date` - дата документа или платежа для событий.
- `amount_semantics = balance_as_of` у РЧБ и соглашений из месячных файлов.
- `amount_semantics = event_amount` у контрактов, платежей и БУАУ.

Для РЧБ snapshot извлекается из заголовка или имени файла. Для соглашений snapshot берется из периода или имени файла. Отчетные даты для UI берутся только из РЧБ и соглашений; одиночные даты платежей не попадают в `/api/catalog/reporting-dates`.

В режиме `as_of` РЧБ и соглашения выбираются по последнему snapshot не позже выбранной даты. Контракты, платежи и БУАУ включаются, если дата события не позже выбранной даты.

## Склейка объектов

Стабильный ключ объекта:

- если код заполнен: `object_key = object_code_norm + "|" + budget_norm`;
- если код пустой: `object_key = "name:" + normalize_name(object_name) + "|" + budget_norm`.

Тот же ключ используется в агрегировании, сравнении и карточке объекта. Если ключ построен по названию, UI показывает предупреждение о возможной неполной связке источников.

## Готовность данных

`/api/readiness` проверяет, попали ли в текущую выборку РЧБ, соглашения, контракты, платежи/БУАУ, есть ли разрывы источников и пустой ли результат. Статусы `ok`, `warn`, `bad` отображаются пользователю простыми формулировками без технических полей.

## Excel и карточка объекта

Excel-экспорт содержит выводы, итоги, агрегированные объекты, проблемные объекты, исходные строки и методику as-of расчета. Карточка объекта открывается по `object_key`, показывает цепочку денег, риск, источники, документы и до 100 исходных строк.
