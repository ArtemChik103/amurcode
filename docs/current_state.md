# Текущее состояние MVP

Проект реализован как компактный in-memory прототип на стандартной библиотеке Python.

## Структура

```text
C:/Users/User/lol/

  README.md
  static/

    4_BUAU_Export/
  docs/
    rag/

```

## Backend

`app.py` загружает данные из `case/`, нормализует записи, фильтрует их, агрегирует суммы, сравнивает периоды и отдаёт JSON API через `ThreadingHTTPServer`.

Добавлены:

- `GET /api/catalog/quick-actions` для быстрых сценариев;
- `GET /api/readiness` для простой проверки полноты данных текущей выборки;
- `GET /api/object` для карточки одного объекта по стабильному `object_key`;
- `GET /api/export.xlsx` для Excel-выгрузки с листами выводов, итогов, объектов, проблем, исходных строк и методики;
- `GET /api/export.pdf` для короткого презентационного PDF-отчета с теми же фильтрами, что Excel;
- `POST /api/assistant` для rule-based и опционального Groq-помощника со строгим action-контрактом;
- `POST /api/explain` для простого объяснения результата без передачи raw records в LLM;
- mini-RAG по markdown-документам из `docs/rag`;
- человекочитаемый `human_summary` в `/api/trace`.

Каждая агрегированная строка получает риск: балл, уровень, пользовательскую подпись и короткое объяснение. `/api/query` возвращает `attention_summary`, а `/api/compare` - `compare_insights`, чтобы фронтенд не собирал управленческий вывод из технических полей. В этих блоках есть `next_actions` для кнопок продолжения работы, а `top_risks` содержит короткое название, код, бюджет и суммы по цепочке денег. Карточка объекта возвращает `diagnosis` с простым списком проверок.

## Frontend

Vue 3 UI через CDN работает без сборки. Первый экран теперь начинается с быстрых действий, единой строки запроса и короткого вывода. Отдельный блок "Спросить помощника" удален. Расширенные фильтры сохранены, но визуально вторичны.

Добавлены:

- быстрый старт;
- сценарий "Демо за 60 секунд" для проблемных СКК;
- единая строка запроса: пользователь пишет объект, код или вопрос, а UI вызывает `/api/assistant` и применяет action;
- smart suggestions как локальный fallback без LLM;
- короткий вывод над результатом;
- блок "Что делать дальше" с кнопками продолжения;
- блок главных рисков;
- простые canvas-графики "Воронка денег" и "Риски";
- компактное раскрываемое объяснение "Как считается" у графика "Риски", скрытое по умолчанию;
- понятная таблица со статусами;
- сортировку проблем по риску и быстрые фильтры критичных/высоких рисков;
- хорошие пустые состояния;
- проверка готовности данных;
- карточка объекта с pipeline, документами и исходными строками;
- основной Excel-экспорт с листом `Выводы`, оформлением отчета, подсветкой рисков, PDF-экспорт для показа руководителю и дополнительный экспорт таблицы.

РЧБ читает лимиты, БО и кассу по смысловым префиксам заголовков, а не по конкретному году в названии колонки. Основная дата отчета остается reporting date из РЧБ и соглашений.

## RAG

`docs/rag/*.md` содержит короткие объяснения терминов, шаблонов, метрик, источников, сценариев и ограничений. Retrieval простой: совпадения слов в локальных markdown-файлах, без векторной БД.

## Assistant

Assistant не считает суммы и не меняет данные. Он выбирает intent, режим, шаблон, метрики, фильтр проблем, дату и вкладку UI. Финансовый результат всегда строится существующими backend-функциями.

## Локальный Groq ключ

Создайте `.env` из `.env.example`. Ключ используется только локально и не коммитится. Без ключа помощник работает по правилам. Для LLM integration tests включите `RUN_LLM_TESTS=1`.

## Ограничения

- Нет БД и миграций.
- Нет фонового импорта.
- Нет векторной базы.
- Groq опционален; без ключа работает fallback по правилам.
