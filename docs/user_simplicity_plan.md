# План упрощения продукта

Целевой пользователь не обязан знать РЧБ, КЦСР, snapshot-даты, источники и метрики. Первый экран предлагает задачи и одну строку запроса: показать СКК, найти объект, сравнить периоды, найти проблемы с исполнением и скачать таблицу.

## Единая строка запроса

Пользователь пишет обычный запрос. Система сама выбирает: быстрый сценарий, rule-based или LLM. Отдельного выбора между "поиском" и "помощником" нет.

## Быстрые сценарии

- Показать СКК.
- Показать КИК.
- Показать 2/3.
- Показать ОКВ.
- Сравнить СКК между первой и последней датой.
- Найти строки, где есть лимиты, но низкое исполнение.
- Демо за 60 секунд.
- Проблемные СКК для демонстрации жюри.

Сценарий "Демо за 60 секунд" сразу открывает вкладку проблем, включает фильтр проблем исполнения по СКК, показывает блок главных рисков и плашку с шагами. Целевой путь:

1. Нажать `Демо за 60 секунд`.
2. Открыть главный риск.
3. Показать `Как считается` у графика рисков, если спросят.
4. Скачать PDF для руководителя.
5. Скачать Excel для детальной проверки.

После любого быстрого сценария блок "Что делать дальше" предлагает следующий клик без ручного ввода: открыть риск, показать проблемы без кассы/оплат/документов, скачать PDF или скачать Excel.

## Доверие к данным

Кнопка проверки показывает простую готовность данных: найдены ли плановые данные, соглашения, контракты, платежи, есть ли разрывы и пуста ли выборка. `Контроль загрузки` показывает строки, источники, предупреждения качества, объекты только по названию и объекты только в одном источнике. Тот же контроль попадает в Excel на лист `Контроль загрузки`.

## Риск простыми словами

Риск показывает, какие объекты стоит проверить первыми. Он растет, если нет кассы, оплат или документов, если касса низкая, данные есть не во всех источниках или сумма плана крупная. Это не юридический вывод, а подсказка для бюджетного специалиста или руководителя.

Блок "Что требует внимания" объясняет ситуацию обычными фразами: сколько объектов без кассы, сколько с документами без оплат, где есть разрывы данных и какая доля кассы от плана. Технические названия полей в этот блок не выводятся.

## Карточка и экспорт

Из таблицы объектов и списка проблем можно открыть карточку объекта: диагноз "Что проверить", цепочка денег, риск, блок `Почему такой риск`, документы и исходные строки. Проверяющий может сохранить статус, ответственного и комментарий; данные проверки хранятся локально в `data/reviews.json`. PDF дает короткий оформленный отчет для руководителя, Excel остается детальной рабочей выгрузкой с листами `Выводы`, `Итоги`, `Объекты`, `Проблемы`, `Исходные строки`, `Контроль загрузки`, `Методика`; экспорт таблицы сохранен как дополнительный быстрый вариант.

## Загрузка новых данных

Пользователь открывает `Расширенные настройки` -> `Загрузить данные`, выбирает источник и файл `.csv` или `.xlsx`. Доступны источники `rcb`, `agreements`, `state_task_contracts`, `state_task_payments`, `buau`. Максимальный размер - 50 MB. Файлы сохраняются локально в `data/uploads/<source_type>/`; при совпадении имени добавляется timestamp suffix. Ошибки показываются из стабильных кодов `invalid_extension`, `source_type_required`, `file_required`, `file_too_large`.

## Офлайн-демо и запуск

Vue runtime лежит локально в `/static/vendor/vue.global.prod.js`, поэтому демо не зависит от CDN. Сервер можно запускать как `python app.py 8000` или напрямую через FastAPI: `uvicorn analytics.api:app --host 127.0.0.1 --port 8000`. `HOST` и `PORT` доступны как переменные окружения.

## Что делает assistant

Assistant понимает пользовательский запрос, выбирает intent, готовый отчёт, режим, метрики, problem filter, дату и вкладку UI. После этого обычный backend считает выборку.

## Что assistant не делает

Assistant не считает суммы, не фильтрует raw records самостоятельно, не меняет данные и не становится источником финансовой истины.

## Fallback без Groq

Если `GROQ_API_KEY` не задан или Groq недоступен, `/api/assistant` возвращает rule-based результат. Основной UI, быстрые действия и умная строка продолжают работать.

## Локальный Groq ключ

Создайте `.env` из `.env.example`. Ключ используется только локально и не коммитится. Без ключа помощник работает по правилам. Для LLM integration tests включите `RUN_LLM_TESTS=1`.

## Проверка перед показом

Перед показом: запустить тесты, открыть локальный сервер, нажать `Демо за 60 секунд`, убедиться что открылась вкладка `Проблемы`, открыть главный риск, проверить `Почему такой риск`, сохранить статус проверки, открыть `Контроль загрузки`, скачать PDF и Excel. На главном экране не должны показываться machine fields: `snapshot`, `trace`, `pipeline`, `problem_reasons`, коды факторов риска.
