LLM-помощник не считает суммы и не меняет исходные данные.

Все суммы считает backend через фильтрацию и агрегацию загруженных записей.

В Groq не отправляются все raw records. Для assistant отправляются только запрос пользователя, список шаблонов, список метрик, доступные даты, быстрые сценарии и короткий RAG-контекст. Для explain отправляются только агрегированные выводы, readiness и top risks без исходных строк.

Если Groq недоступен или ключ не задан, помощник работает по правилам.

Риск в интерфейсе - это приоритет проверки, а не юридический вывод. Текущая версия методики: `RISK_MODEL_VERSION = "2026-05-05"`. Основные факторы: нет кассы, нет оплат, нет документов, низкая касса, разрыв данных между источниками, крупный план.

Проверочные статусы сохраняются только локально в `data/reviews.json`. Загруженные через UI файлы сохраняются только локально в `data/uploads/<source_type>/`; поддерживаются `.csv` и `.xlsx` до 50 MB для источников `rcb`, `agreements`, `state_task_contracts`, `state_task_payments`, `buau`.

Vue runtime лежит локально в `/static/vendor/vue.global.prod.js`, поэтому демо не требует CDN. Сервер запускается командой `python app.py 8000` или `uvicorn analytics.api:app --host 127.0.0.1 --port 8000`.

## Локальный Groq ключ

Создайте `.env` из `.env.example`. Ключ используется только локально и не коммитится. Без ключа помощник работает по правилам. Для LLM integration tests включите `RUN_LLM_TESTS=1`.
